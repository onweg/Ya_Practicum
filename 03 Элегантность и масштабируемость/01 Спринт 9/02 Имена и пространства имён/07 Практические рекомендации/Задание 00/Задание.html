Задание

В этой задаче нужно улучшить код другого программиста, применив пространства имён.

В заготовке кода — библиотека Statistics для вычисления следующих статистик набора чисел:

  • сумма,
  • максимум,
  • среднее арифметическое,
  • стандартное отклонение,
  • мода — наиболее часто встречаемое значение.

Вычислитель каждой из статистик — класс в программе.

Измените программу следующим образом:

  1. все классы, функции и тесты библиотеки Statistics поместите в namespace statistics,
  2. все вычислители статистик перенесите во вложенное пространство имён statistics::aggregations,
  3. все тесты перенесите в statistics::tests,
  4. переименуйте классы из statistics::aggregations так, как сказано ниже,
  5. переименуйте тесты, убрав общий префикс TestStat, но оставив Aggreg,
  6. уберите using namespace std; из h-файлов,
  7. поместите GetPrinterValue в пространство имён statistics::tests::detail.

В самом statistics помимо вложенных пространств имён tests и aggregations остаётся только класс AggregPrinter.

Классы вычислителей нужно переименовать так:

  • SumAggregation → Sum
  • AggregateMaximum → Max
  • AggregatorAverage → Mean
  • AggregStd → StandardDeviation
  • Mode переименовывать не нужно

Несмотря на то, что Aggreg — также общий префикс для тестов, убирать его не нужно, поскольку из названия пространства имён тестов statistics::tests не следует, что это тест агрегирующего вычислителя. Можно было вынести тесты вычислителей в отдельное пространство имён, но в этом задании все тесты библиотеки расположены в одном.

Структура имён получается такая:

  • statistics
    • aggregations
      • Sum
      • Max
      • Mean
      • StandardDeviation
      • Mode
    • tests
      • detail
        • GetPrinterValue
      • AggregSum
      • AggregMax
      • AggregMean
      • AggregStandardDeviation
      • AggregMode
      • AggregPrinter
    • AggregPrinter

Ограничения

  • Не меняйте логику работы вычислителей и тестов.
  • Все классы и функции должны называться так, как сказано в условии, и находиться в нужных пространствах имён.
  • Не забудьте убрать using namespace std; из h-файла.
  • В пространствах имён statistics и statistics::aggregations запрещается использовать using namespace std, но using namespace std::literals — разрешён.

Как будет проверяться ваша программа

Мы проверим, что:

  • присутствуют все требующиеся имена и они находятся в нужных пространствах имён,
  • классы, функции и тесты работают правильно,
  • в h-файле нет using namespace std,
  • вы не используете using namespace std в других пространствах имён.

Учтите: при неправильной конфигурации имён возникнет ошибка компиляции.

Подсказка

  • Следуйте схеме имён, указанной в условии.
  • В пространстве имён statistics::tests используйте using namespace aggregations, чтобы не прибегать каждый раз к квалификации. При этом могут возникнуть конфликты.
