Умный указатель shared_ptr

В прошлом уроке вы познакомились с умным указателем unique_ptr, который единолично владеет объектом в динамической памяти. Он удобен в тех случаях, когда время жизни одного объекта ограничено временем жизни другого объекта-владельца. Примеры:

  • Время жизни дочерних узлов двоичного дерева ограничено временем жизни их родительского узла.
  • Время жизни щупалец осьминога ограничено временем жизни осьминога.

Однако встречаются задачи, где конкретный объект должен жить, пока в программе есть хотя бы один объект-владелец, ссылающийся на него. Указатель unique_ptr в этом случае бессилен, ведь в программе может быть только один его экземпляр, ссылающийся на конкретный объект.

В таких ситуациях на помощь приходит умный указатель std::shared_ptr.

Указатель shared_ptr

shared_ptr — умный указатель, обеспечивающий совместное владение динамически выделенным ресурсом. Несколько экземпляров shared_ptr могут владеть одним и тем же объектом. Объект разрушается использованием delete, а память освобождается, когда:

  • вызывается деструктор последнего экземпляра shared_ptr, владевший этим объектом;
  • последнему экземпляру shared_ptr, владевшему этим объектом, присвоено значение другого указателя. Для этого использована операция присваивания или вызов метода reset.

Можно создать shared_ptr на основе сырого указателя на объект в куче:

shared_ptr<Object> obj_ptr{new Object()};

Либо сразу создать объект в куче и получить владеющий этим объектом shared_ptr, вызвав функцию make_shared:

auto obj_ptr = make_shared<Object>();

Рассмотрим принципы работы shared_ptr на примере совместного владения Собаками:

class Dog {
public:
    explicit Dog(string name)
        : name_(move(name))  //
    {
        cout << name_ << " dog has been created"sv << endl;
    }

    ~Dog() {
        // Все псы попадают в рай
        cout << name_ << " dog has gone to heaven"sv << endl;
    }

private:
    string name_;
};

Люди могут владеть Собаками, используя указатель shared_ptr. Для простоты будем считать, что каждый человек в один момент времени владеет только одной собакой. Несколько человек может владеть одной собакой — например, если они живут вместе.

class Person {
public:
    explicit Person(string name)
        : name_(move(name)) {
    }

    ~Person() {
        cout << name_ << " has died"sv << endl;
    }

    // Стать владельцем собаки
    void OwnDog(shared_ptr<Dog> dog) {
        dog_ = move(dog);
    }

    const shared_ptr<Dog>& GetDog() const {
        return dog_;
    }

    // Перестаём владеть собакой
    void ReleaseDog() {
        dog_.reset();
        assert(!dog_);  // Указатель обнуляется после вызова reset
    }

private:
    string name_;
    shared_ptr<Dog> dog_;
};

Сначала создаются объекты maria и ivan, а затем, во вложенном блоке, умные указатели dog1, dog2 и dog3, ссылающиеся на Белку, Стрелку и Хатико. Собаки Белка и Стрелка передаются во владение Ивану и Марии:

int main() {
    // Жили-были Иван да Марья
    Person maria("Maria"s);
    Person ivan("Ivan"s);

    {
        // У Ивана была собака Белка
        shared_ptr<Dog> dog1(new Dog("Belka"s));
        ivan.OwnDog(dog1);

        // А у Марьи - Стрелка
        auto dog2 = make_shared<Dog>("Strelka"s);
        maria.OwnDog(dog2);

        // А вот у Хатико не было хозяина
        auto dog3 = make_shared<Dog>("Hatiko"s);

        // Прошли годы
        cout << "-----"sv << endl;
        cout << "Several years have passed"sv << endl;
    }
    ...

При выходе из внутреннего блока единственный указатель dog3, ссылающийся на Хатико, разрушается, а вместе с ним и соответствующий объект. Разрушаются также указатели dog1 и dog2, однако собаки Белка и Стрелка продолжают существовать, ведь на них всё ещё ссылаются указатели Ивана и Марьи:

Belka dog has been created
Strelka dog has been created
Hatiko dog has been created
-----
Several years have passed
Hatiko has gone to heaven

Затем Иван берёт в совместное владение собаку Марии, одновременно переставая владеть своей собакой Белкой:

    cout << "-----"sv << endl;

    // Стал Иван вместе с Марьей владеть её собакой
    cout << "Ivan owns Maria's dog"sv << endl;
    ivan.OwnDog(maria.GetDog());
    cout << "-----"sv << endl;

Так как единственному оставшемуся указателю на Белку было присвоено другое значение, Белка тоже отправляется на небеса:

-----
Ivan owns Marias's dog
Belka dog has gone to heaven
-----

Затем Мария перестаёт владеть Стрелкой:

    // Ушла Марья от Ивана, оставив ему собаку
    cout << "Maria leaves her dog"sv << endl;
    maria.ReleaseDog();
    cout << "-----"sv << endl;

Так как у Стрелки остаётся еще один владеющий указатель со стороны Ивана, с ней ничего не происходит:

Maria leaves her dog
-----

В конце функции main из своей области видимости выходят локальные переменные ivan и maria:

    // Жили они долго и счастливо и умерли в один день
}

Сначала разрушается переменная ivan, а вместе с ним — последний указатель на Стрелку. Затем разрушается переменная maria:

Ivan has died
Strelka dog has gone to heaven
Maria has died

Таким образом, shared_ptr позволяет динамически продлевать время жизни объекта в куче, пока у этого объекта есть хотя бы один владелец. Как только все владеющие объектом указатели разрушаются или перестают указывать на него, объект удаляется.

shared_ptr полезен, когда несколько объектов с разным временем жизни совместно используют некоторый ресурс. Этот ресурс нельзя удалять, пока он используется хотя бы одним объектом. Чтобы ресурс не занимал впустую память, его нужно удалить, когда в нём больше никто не нуждается. shared_ptr автоматизирует управление временем жизни такого ресурса.

Например, в программе для создания презентаций несколько фигур могут совместно использовать одну и ту же текстуру для рисования своей внутренней области.

На рисунке показан слайд презентации с тремя фигурами. Фигура №1 использует текстуру A, а фигуры №2 и №3 — текстуру B. Время жизни фигур на слайде заранее не известно, так как пользователь может удалить в редакторе любую из них в произвольный момент. При этом нужно своевременно удалить из памяти текстуру, если не осталось использующих её фигур.

Текстуру A можно удалить из памяти программы сразу после удаления фигуры №1, а текстуру B — только после удаления фигур №2 и №3. Текстуры играют роль ресурсов, совместно используемых фигурами. Управление этими ресурсами легко автоматизировать, если фигуры будут ссылаться на свои текстуры, используя shared_ptr. Дубликат фигуры будет использовать shared_ptr, ссылающийся на ту же текстуру, что и оригинальная фигура. Каждая текстура будет автоматически удалена, когда на слайде не останется ни одной использующей её фигуры.

Создание shared_ptr из unique_ptr

shared_ptr имеет специальный конструктор, принимающий unique_ptr по rvalue-ссылке. Используя его, можно переместить содержимое unique_ptr в shared_ptr, превратив единоличное владение объектом в совместное:

auto u_ptr = std::make_unique<Dog>("Belka"s);
...
shared_ptr<Dog> s_ptr{std::move(u_ptr)};
// u_ptr пуст - право владения объектом передано указателю s_ptr
assert(!u_ptr);

Обратное преобразование из shared_ptr в unique_ptr невозможно: основная задача shared_ptr — совместное владение, что противоречит идее уникального владения. Поэтому если вам нужно написать функцию, которая будет создавать объект и возвращать умный указатель, возвращайте unique_ptr, а затем присваивайте его shared_ptr, когда требуется. Возвращать shared_ptr стоит тогда, когда вы точно знаете, что возвращаемый объект предназначен лишь для совместного владения.
