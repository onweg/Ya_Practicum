Какими свойствами должен обладать метод EmplaceBack класса Vector? Выберите несколько вариантов.

  • (-) Метод не должен выбрасывать исключений.

Вектору может не хватить места в куче, а конструкторы его элементов могут выбрасывать исключения. Поэтому требовать от EmplaceBack гарантию отсутствия исключений не получится.

  • (+) Метод должен уметь принимать любое количество аргументов произвольного типа.

Вектор — универсальный класс. Его элементами могут быть классы, конструкторы которых различаются количеством и типом аргументов.

  • (-) Метод должен принимать свои аргументы по значению.

Если EmplaceBack станет принимать аргументы по значению, они будут лишний раз копироваться. Кроме того, возникнут проблемы с передачей аргументов, не имеющих копирующего конструктора.

  • (-) Метод должен принимать свои аргументы по ссылке.

Тогда не получится передать конструктору T временные объекты и константы.

  • (-) Метод должен принимать свои аргументы по константной ссылке.

Тогда для перемещаемых аргументов конструктора возникнут лишние копирования.

  • (-) Метод должен принимать свои аргументы по rvalue-ссылке.

Тогда не получится передать конструктору T параметр-константу.

  • (+) Метод должен принимать свои аргументы по Forwarding-ссылке.

В этом случае он сможет эффективно передавать конструктору T константные и неконстантные ссылки, а также ссылки на временные объекты.

  • (-) Метод должен выполняться за время O(1/N).

Здравствуйте, уважаемый Чак Норрис.

  • (+) Метод должен выполняться за время O(1).

Подобно методу PushBack временнáя сложность EmplaceBack должна быть амортизированной константой.

  • (-) Метод должен выполняться за время O(logN).

Это слишком долго. Вставку в конец вектора, увеличивающего свою вместимость вдвое, можно реализовать за амортизированное константное время.

  • (-) Метод должен выполняться за время O(N).

Это слишком долго. Вставку в конец вектора, увеличивающего свою вместимость вдвое, можно реализовать за амортизированное константное время.

Есть ли необходимость в обоих методах PushBack, если EmplaceBack может принимать аргументы T&, const T& и T&&?

Vector<Cats> cats;

Cat cat_1("Tom", 3);
Cat cat_2("Leopold", 4);

cats.EmplaceBack(cat_1);             // Такой же результат даёт вызов cats.PushBack(cat_1);
cats.EmplaceBack(std::move(cat_2));  // Такой же результат даёт вызов cats.PushBack(std::move(cat_2));

  • (-) Методы PushBack больше не нужны. Можно смело заменить их на EmplaceBack.

  • (+) Методы PushBack по-прежнему могут быть полезны.
