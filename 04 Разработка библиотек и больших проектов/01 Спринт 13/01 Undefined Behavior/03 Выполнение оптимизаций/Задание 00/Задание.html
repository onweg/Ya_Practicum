Задание

Напишите программу, которая складывает два целых числа и корректно обрабатывает переполнение при сложении.

Программа должна считать из входного потока два целых числа и вывести в выходной поток их сумму. Гарантируется, что оба числа на входе помещаются в тип std::int64_t. Если результат их сложения выходит за пределы типа std::int64_t, в выходной поток вместо суммы чисел нужно вывести строку Overflow!.

Примеры входных и выходных данных

Ввод:

5 -3

Вывод:

2

Ввод:

4567890123456789012 5678901234567890123

Вывод:

Overflow!

Как будет тестироваться ваш код

Программа будет собрана с ключом -fsanitize=undefined, который вставляет в код дополнительные проверки. Они нужны, чтобы обнаружить неопределённое поведение. Если оно будет выявлено, результат не засчитается. С этим флагом вы познакомитесь ближе в следующих уроках.

Подсказка

  • Переполнение при сложении двух целых чисел возможно, только когда оба числа положительные либо когда оба числа отрицательные. В первом случае результат может превысить максимально допустимое значение, а во втором — минимально допустимое. При сложении двух чисел с разными знаками переполнение возникнуть не может, так как результат окажется между ними.
  • Переполнение при вычитании двух целых чисел возможно, только когда оба числа имеют разные знаки. Ведь в этом случае результат аналогичен сложению или вычитанию чисел с одним и тем же знаком: a-b эквивалентно a+(-b).
  • Чтобы получить минимальное и максимальное значение типа int64_t, используйте макросы INT64_MIN и INT64_MAX, которые объявлены в файле <climits>. Также можно взять шаблонные функции std::numeric_limits<T>::min() и std::numeric_limits<T>::max(). Второй способ удобен, если хотите написать универсальную шаблонную функцию для безопасного сложения целых чисел произвольной разрядности. Например, такую:
template <typename T>
optional<T> SafeAdd(T a, T b) {
    // при переполнении возвращается nullopt, иначе - сумма
    ...
}
  • При проверке условия a + b > MAX при положительных a и b возможно переполнение при сложении a+b. Это условие можно переписать как a > MAX - b. В отличие от изначального варианта выражение MAX-b не вызовет переполнения при положительном b. Аналогично можно избавиться от переполнения при проверке условия a + b < MIN, когда a и b — отрицательные.
