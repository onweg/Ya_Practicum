Библиотеки: какие они бывают

Многие любят велосипеды. Но только программисты любят их писать — ведь интереснее написать код самому, чем использовать чужой. Иногда «велосипед» оказывается слишком сложным, и тогда всё-таки приходится использовать библиотеку. Программисту на Python в этом случае достаточно выполнить команду pip install, а затем добавить в свою программу одну строчку импорта. В C++ всё сложнее.

Библиотека может распространяться как в виде исходного кода, так и в виде уже собранных бинарных файлов. В первом случае всё просто — вы добавляете код библиотеки в свой проект. Скажем, header-only библиотеки, которые состоят только из h-файлов, вообще не надо собирать — достаточно добавить нужные #include. Пример — большая и производительная библиотека линейной алгебры Eigen.

В чём плюсы распространения библиотеки в виде бинарных файлов? Выберите один или несколько вариантов.

  • Не нужно тратить время на сборку библиотеки — компиляцию и компоновку — при её использовании.

  • Производитель может держать исходный код в секрете.

  • Бинарные файлы проще читать.

В чём минусы распространения библиотеки в виде бинарных файлов? Выберите один или несколько вариантов.

  • Вместо единого исходного кода приходится распространять множество сборок и регулярно их обновлять.

  • Шаблонные классы и функции, реализации которых распространяются в собранном виде, нельзя настроить на ваш тип.

  • Бинарные файлы труднее читать.

Использование бинарных библиотек различается в зависимости от операционной системы и компилятора. Но везде возможны два варианта:

  • Динамическая, или разделяемая библиотека. Код этой библиотеки не встраивается в приложение, поэтому она должна распространяться вместе с программой.
  • Статическая библиотека. Код этой библиотеки компонуется вместе с объектными файлами вашей программы.

Предположим, библиотека распространяется в виде бинарных файлов. Программа вызывает функцию библиотеки. Как вы знаете, при вызове функции должна быть доступна её сигнатура. Откуда компилятор может взять сигнатуру функции в этом случае?

  • Он может взять сигнатуру из бинарного файла библиотеки.

  • Вместе с библиотекой должны распространяться h-файлы, содержащие все необходимые сигнатуры.

  • Функцию библиотеки нельзя вызвать.

Распространять только бинарные файлы недостаточно. Ведь они могут быть использованы только при компоновке, когда компилятор уже завершил свою работу. Компилятор тоже должен знать о библиотеке, и другого способа, кроме include-файлов, нет.

Библиотека C++ — это пакет, содержащий как бинарные, так и h-файлы. Некоторые библиотеки включают в себя и динамическую, и статическую версии, предоставляя разработчику выбор, а некоторые только одну из них. Типичная библиотека имеет следующие каталоги или категории файлов:

  • include — содержит h-файлы, подключаемые вашей программой через #include;
  • lib — бинарные файлы, компонуемые с вашей программой;
  • dll (под Windows) — динамические библиотеки;
  • framework (под macOS) — пакет, содержащий в себе бинарные файлы и ресурсы.
