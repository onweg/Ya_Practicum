Задание

В уроке приведён способ бинарной сериализации/десериализации:

output_stream.write(reinterpret_cast<const char*>(&x), sizeof(x));
input_stream.read(reinterpret_cast<char*>(&x), sizeof(x));

Но делать так неудобно:

  • код выглядит довольно громоздким;
  • имя переменной используется дважды;
  • использование reinterpret_cast и сырых указателей небезопасно и нежелательно, с ними легко допустить ошибку.

Разработайте библиотеку для выполнения бинарной сериализации и десериализации, которая помимо простых типов сможет работать с контейнерами.

Библиотека должна уметь сериализовывать составные контейнеры, например std::vector<std::map<int, std::string>>.

Сериализация осуществляется функциями Serialize, принимающими константную ссылку на значение и поток ostream. Десериализация — функциями Deserialize, принимающими поток istream и неконстантную ссылку.

Всего нужно реализовать четыре варианта функций Serialize и Deserialize:

  • template <typename T> void Serialize(T pod, std::ostream& out), template <typename T> void Deserialize(std::istream& in, T& pod) — должны работать со скалярными типами и простыми структурами, содержащими только скалярные типы и такие структуры. Иными словами, для типов с конструктором по умолчанию, значение которых определяется памятью объекта. Эти методы не обязаны корректно работать с типами, содержащими ссылки или указатели.
  • void Serialize(const std::string& str, std::ostream& out), void Deserialize(std::istream& in, std::string& str) — должны работать со строками произвольной длины. Сначала в поток пишется длина, затем сама строка. Дополнительный нулевой байт записывать не надо.
  • template <typename T> void Serialize(const std::vector<T>& data, std::ostream& out), template <typename T> void Deserialize(std::istream& in, std::vector<T>& data) — работа с вектором по аналогии со string. Тип T может быть любым допустимым для Serialize типом.
  • template <typename T1, typename T2> void Serialize(const std::map<T1, T2>& data, std::ostream& out), template <typename T1, typename T2> void Deserialize(std::istream& in, std::map<T1, T2>& data) — функции для работы с map. Они пишут размер контейнера, а затем последовательно пишут все элементы в том порядке, в котором они расположены в контейнере. При сериализации пары сначала нужно записать ключ, затем — значение.

Формат входных данных

Функции Deserialize на вход подаётся поток, который содержит представление, полученное ранее функцией Serialize.

Формат выходных данных

Сериализация в функциях Serialize должна выполняться так:

  • Значение неизвестного типа сериализуется по аналогии с уроком.
  • При сериализации любого контейнера сначала записывается число типа size_t — размер контейнера, — а затем уже сами элементы.
  • Для std::map записывайте элементы как пары, состоящие из ключа и значения.

Ограничения

Функции Deserialize должны восстанавливать исходный объект по сериализованному представлению.

Пример

Пусть требуется сериализовать такой словарь:

std::map<uint32_t, std::string> m = {
    {1, "hello"},
    {2, "bye"},
};

На 64-битной платформе должен получиться следующий результат:

00000000  02 00 00 00 00 00 00 00  01 00 00 00 05 00 00 00  |................|
00000010  00 00 00 00 68 65 6c 6c  6f 02 00 00 00 03 00 00  |....hello.......|
00000020  00 00 00 00 00 62 79 65                           |.....bye|

Бинарный код зависит от порядка байтов вашей платформы. Тут показан код с порядком от младшего к старшему. Скорее всего, на вашем компьютере он такой же. Вывод получен утилитой hd, печатающей бинарный код.

Ещё один пример есть в заготовке кода.

Что отправлять на проверку

Реализуйте все функции Serialize и Deserialize, декларации которых представлены в заготовке. Поскольку почти все функции шаблонные, реализации напишите прямо в заголовочном файле saveload.h. Нешаблонные функции нужно обозначить как inline — это обязательное требование для функций, реализованных в заголовочном файле.

Как будет тестироваться ваш код

Мы проверим, что:

  • программа правильно сериализует скалярные типы;
  • программа правильно сериализует пользовательские типы;
  • программа правильно сериализует контейнеры;
  • программа правильно сериализует составные контейнеры;
  • программа правильно десериализует всё вышеперечисленное.

Подсказка

  • При сериализации и десериализации общего типа воспользуйтесь способом из урока. Понадобится reinterpret_cast и операция взятия адреса.
  • При сериализации и десериализации значения string используйте метод data.
  • Размеры контейнеров и их элементы сериализуйте уже готовым вариантом Serialize.
