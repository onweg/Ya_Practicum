Protobuf — знакомство и установка

Познакомьтесь — Google Protocol Buffers, можно просто Protobuf. Широко применяемый протокол бинарной сериализации. Поддерживает самые разные языки, например C++, Python, Java и JavaScript. Данные могут быть сериализованы программой на C++, переданы по сети и на другом конце прочитаны программой на Python.

Знакомимся чуть ближе — proto-файлы

Формат описывается в файле с расширением .proto. Их ещё называют proto-схемами. Этот файл подаётся специальному компилятору, который создаст код на выбранном языке программирования, способный сохранять, читать и менять данные указанного формата. Как можно догадаться, выбирать будем язык C++.

Рассмотрим пример proto-файла, который будет полезен при разработке адресной книги:

syntax = "proto3";

enum Gender {
    MALE = 0;
    FEMALE = 1;
    UNSPECIFIED = 2;
}

message Address {
    string street = 1;
    uint32 building = 2;
}

message Person {
    string name = 1;
    int32 birth_year = 2;
    Address address = 3;
    repeated string phone_numbers = 4;
    Gender gender = 5;
}

Proto-файл определяет сообщение — совокупность значений, аналог структуры в C++. Основа адресной книги — данные о человеке, которые кодируются сообщением Person. Оно включает такую информацию, как имя, год рождения, адрес, список телефонных номеров и пол. Адрес сам по себе имеет структуру, поэтому кодируется отдельным сообщением.

Если запустить компилятор Protobuf, чтобы сгенерировать C++ код из этого файла, для каждого сообщения будет создан отдельный класс. Например, подобный класс получится для сообщения Person:

class Person ... {
...
public:
    const std::string& name() const;
    void set_name(const std::string& value);
...
    ::PROTOBUF_NAMESPACE_ID::int32 age() const;
    void set_age(::PROTOBUF_NAMESPACE_ID::int32 value);
...
};

Сгенерированными классами можно пользоваться как обычными классами C++.

Знакомимся лично — устанавливаем Protobuf

Скачать Protobuf можно с репозитория на GitHub. Выберите архив protobuf-cpp с исходным кодом последней версии и распакуйте его на своём компьютере. Исходный код содержит CMake-проект.

Создадим папки build-debug и build-release для сборки двух конфигураций Protobuf. Если вы используете Visual Studio, будет достаточно одной папки build. А если CLion или QT Creator, IDE автоматически создаст папки при открытии файла CMakeLists.txt. Visual Studio тоже умеет открывать CMakeLists.txt непосредственно, но проект можно будет использовать исключительно для сборки, а не для работы над исходным кодом. Впрочем, для наших целей этого достаточно.

Заранее создадим папку, в которой разместим пакет Protobuf. Будем называть её /path/to/protobuf/package.

Если вы собираете не через IDE, в папке build-debug выполните следующие команды:

cmake <путь к protobuf>/cmake -DCMAKE_BUILD_TYPE=Debug \
      -Dprotobuf_BUILD_TESTS=OFF \
      -DCMAKE_INSTALL_PREFIX=/path/to/protobuf/package
cmake --build .

Мы задали дополнительные параметры:

  • -Dprotobuf_BUILD_TESTS=OFF — чтобы не тратить время на сборку тестов,
  • -DCMAKE_INSTALL_PREFIX=/path/to/protobuf/package — чтобы сообщить, где нужно будет создать пакет Protobuf.

Для Visual Studio команды немного другие. Конфигурация указывается не при генерации, а при сборке:

cmake <путь к protobuf>/cmake ^
      -Dprotobuf_BUILD_TESTS=OFF ^
      -DCMAKE_INSTALL_PREFIX=/path/to/protobuf/package ^
      -Dprotobuf_MSVC_STATIC_RUNTIME=OFF
cmake --build . --config Debug

Задать переменную protobuf_MSVC_STATIC_RUNTIME при сборке под Visual Studio очень важно. Дело в том, что Visual Studio имеет два набора Runtime-библиотек — библиотек, которые компонуются с каждой собираемой программой. Если окажется, что два бинарных файла зависят от библиотек разных наборов, получите головную боль из-за криптографических ошибок во время компоновки приложения. Типа такой:

mismatch detected for 'RuntimeLibrary': value 'MTd_StaticDebug' doesn't match value 'MDd_DynamicDebug'

Пока идёт сборка, вы можете попить чай, посмотреть любимый сериал или поиграть в настольный теннис. Некоторые C++-библиотеки могут собираться часами. Хотя если у вас не IBM PC XT 286, компилятор должен управиться с Protobuf быстрее.

Итак, библиотека готова. Теперь запустим волшебную команду:

cmake --install .

Под Visual Studio нужно указать конфигурацию, поскольку она не задавалась во время генерации:

cmake --install . --config Debug

И CMake скопирует все необходимые файлы в заранее подготовленное место /path/to/protobuf/package. Посмотрим, что же сделал CMake. Структура папки должна получиться такой:

  • bin
    • Тут будет исполняемый файл — protoc или protoc.exe под Windows. Это и есть компилятор proto-файлов.
  • lib
    • Здесь статические библиотеки — lib*.a. При сборке под Visual Studio — lib*.lib.
    • pkgconfig
      • Описание библиотеки для программы pkg-config, которая автоматически конфигурирует сборку библиотек C++.
  • include
    • Здесь будут include-файлы.

Проделываем те же шаги с конфигурацией Release. Устанавливать будем в ту же папку. Так как статические библиотеки, собранные в конфигурации Debug, имеют суффикс d, конфликта не возникнет.

Пакет готов, пора применить его на практике!
