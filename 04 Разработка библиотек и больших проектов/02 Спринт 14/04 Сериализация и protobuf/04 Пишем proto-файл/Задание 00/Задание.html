Задание

Напишите proto-файл для нужд библиотеки. Той самой, с книгами, а не то, что вы подумали.

Он должен определять следующие сообщения.

  • Сообщение Author — писатель. Имеет поля:
    • строка first_name — имя автора,
    • строка last_name — фамилия автора,
    • целое неотрицательное birth_year — год рождения автора.
  • Сообщение Publishing — издательство. Имеет поля:
    • строка name — название,
    • строка address — адрес,
  • Сообщение Date — дата. Имеет поля:
    • целое неотрицательное year — год,
    • целое неотрицательное month — месяц,
    • целое неотрицательное day — день.
  • Сообщение User — читатель. Имеет поля:
    • целое неотрицательное id — идентификатор,
    • строка name — полное имя,
    • дата registration_date — дата регистрации.
  • Сообщение Book — книга. Имеет поля:
    • издательство publishing,
    • массив авторов author,
    • целое неотрицательное year — год издания,
    • строка name — название,
    • дата borrow_date — дата последнего получения книги,
    • дата return_date — крайний срок возврата,
    • массив целых неотрицательных holder — идентификаторы читателей, ранее бравших книгу.
  • Сообщение Library — библиотека. Имеет поле:
    • массив книг book,
    • массив читателей reader.

Ограничения

Называйте сообщения и поля в точности так, как это описано в задании.

Что отправлять на проверку

Напишите полностью proto-файл со всеми описанными в задании сообщениями и полями.

Как будет тестироваться ваш код

Ваш proto-файл добавится к программе, использующей описанные поля. Если что-нибудь сделано неверно, возникнет ошибка компиляции.

Подсказка

  • Пишите proto-файл по аналогии с тем, как это сделано в уроке.
  • Для массивов используйте слово repeated.
