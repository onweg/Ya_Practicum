Какая из последовательностей токенов соответствует данной программе?

a = a + 2
print a, "hello"
  • (-) <char,'a'> <char,'='> <char,'a'> <char,'+'> <char,'2'> <newline> <print> <char,'a'> <char,','> <string,"hello"> <newline> <eof>

a — не просто символ, это идентификатор, а 2 — числовая константа.

  • (-) <id,"a"> <char,'='> <id,"a"> <char,'+'> <number,2> <print> <id,"a"> <char,','> <string,"hello"> <eof>

Символ новой строки в Python — важный элемент программы на Mython, поэтому не должен отбрасываться лексическим анализатором. Сравните:

# программа выведет символ конца строки, а потом вызовет object.method()
print
object.method()

и

# программа выведет значение, возвращённое object.method() и символ конца строки
print object.method()
  • (+) <id,"a"> <char,'='> <id,"a"> <char,'+'> <number,2> <newline> <print> <id,"a"> <char,','> <string,"hello"> <newline> <eof>

Верно. Здесь a — идентификатор, 2 — числовая константа, а print — самостоятельный токен, играющий особую роль в работе программы.

  • (-) <id,"a"> <char,'='> <id,"a"> <char,'+'> <number,2> <newline> <id,"print"> <id,"a"> <char,','> <string,"hello"> <newline> <eof>

Лучше сделать print не идентификатором, а отдельной лексемой. Это справедливо и при обработке остальных ключевых слов языка.

Какое из представленных ниже синтаксических деревьев соответствует этому фрагменту программы на языке Mython?

print a + b * c + d

Выберите один или несколько вариантов.

  • (-) Вариант 1

Это дерево соответствует команде print (a+b)*c+d.

  • (+) Вариант 2

Это дерево соответствует команде команде print (a+(b*c))+d, что эквивалентно исходной программе.

  • (-) Вариант 3

Это дерево соответствует команде print (a+b)*(c+d).

  • (-) Вариант 4

Это дерево соответствует команде print (a+d)+(b*c). Эта команда будет эквивалентна исходной, только когда a, b, c и d — числа. Если a — экземпляр класса с переопределённым методом __add__, изменение в порядке выполнения операций может повлиять на результат вычисления выражения.

  • (-) Вариант 5

Это дерево соответствует команде print a + ((b*c) + d). Для чисел результат будет совпадать с ожидаемым, но для объектов с переопределённым методом __add__ разница в работе метода может оказаться существенной.
